# Screenshots (capture guide)

Add PNG or WebP captures here when the app is deployed or running locally. Suggested filenames for submission packs:

| File | What to capture |
|------|-----------------|
| `01-landing.png` | `/` — hero, name + booking code paths, header/footer |
| `02-chat.png` | `/chat` — conversation + agent panel (summaries + optional expanded technical details) |
| `03-chat-voice.png` | `/chat` — mic states (optional; Chrome/Edge) |
| `04-admin-dashboard.png` | `/admin` — Dashboard tab with graphs |
| `04b-admin-agent-log.png` | `/admin` → Agent Activity Log — columns per agent, PM summaries (optional) |
| `05-admin-pulse.png` | `/admin` — Pulse management (preview + actions) |
| `06-admin-bookings.png` | `/admin` — Bookings table + email actions |
| `07-secure-booking.png` | `/secure/GRW-XXXX` — verified booking + form |
| `08-subscribers.png` | `/subscribers` — signup success or duplicate state |

**Tip:** Use the same `NEXT_PUBLIC_BACKEND_URL` as production when capturing so admin and secure flows hit real data.
